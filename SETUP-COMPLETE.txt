🚀 HOW TO GET STARTED (5 MINUTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Start Services
┌─────────────────────────────────────────────────────────┐
│ $ docker-compose up --build                             │
│                                                         │
│ Waits for:                                              │
│   • Ollama (downloads Gemma:7b model - 2-5 min first)   │
│   • Qdrant (ready ~10 sec)                              │
│   • Backend (ready ~5 sec)                              │
│   • Frontend (ready ~2 sec)                             │
└─────────────────────────────────────────────────────────┘

STEP 2: Ingest Documents
┌─────────────────────────────────────────────────────────┐
│ $ docker exec rag-backend python \                       │
│   scripts/ingest_sample_documents.py --sample            │
│                                                         │
│ Ingests 4 sample docs (~20 chunks total):               │
│   • Machine Learning Basics                             │
│   • Vector Databases Guide                              │
│   • RAG Systems Explained                               │
│   • Azure Fundamentals                                  │
└─────────────────────────────────────────────────────────┘

STEP 3: Test the System
┌─────────────────────────────────────────────────────────┐
│ $ docker exec rag-backend python \                       │
│   scripts/test_rag_e2e.py                               │
│                                                         │
│ Tests:                                                  │
│   ✓ Collection setup                                    │
│   ✓ Document embedding                                  │
│   ✓ 5 practice queries                                  │
│   ✓ Results exported to test_results.json               │
└─────────────────────────────────────────────────────────┘

STEP 4: Open Chat Interface
┌─────────────────────────────────────────────────────────┐
│ Open: http://localhost:3000                             │
│                                                         │
│ Try asking:                                             │
│   • "What is machine learning?"                         │
│   • "How does RAG work?"                                │
│   • "What are vector databases?"                        │
│                                                         │
│ System responds with answers + source documents         │
└─────────────────────────────────────────────────────────┘

🔧 CONFIGURATION (Edit .env to Change)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Option A: Local Ollama (Current Default)
┌─────────────────────────────────────────────────────────┐
│ LLM_PROVIDER=ollama                                     │
│ EMBEDDING_PROVIDER=ollama                               │
│ OLLAMA_MODEL=qwen2.5:0.5b                               │
│                                                         │
│ ✓ Free, no API keys                                     │
│ ✓ Fast (1-3 sec per query after loading)                │
│ ✓ Can try other models: mistral, llama2, neural-chat    │
└─────────────────────────────────────────────────────────┘

Option B: Cloud LLM (Groq - Free)
┌─────────────────────────────────────────────────────────┐
│ LLM_PROVIDER=litellm                                    │
│ LITELLM_GROQ_API_KEY=your-key                           │
│ LITELLM_LLM_MODEL=groq/mixtral-8x7b-32768               │
│                                                         │
│ Get free API key: https://console.groq.com              │
│ ✓ Faster inference                                      │
│ ✓ Better model quality                                  │
│ ✓ Free tier available                                   │
└─────────────────────────────────────────────────────────┘


📚 COMPLETE USE CASES

USE CASE 1: Process Your Own Documents
├─ Place docs in ./docs/
├─ Run: docker exec rag-backend python scripts/ingest_sample_documents.py --file /app/docs/mydoc.md
└─ Chat interface automatically uses your content

USE CASE 2: Build Knowledge Base
├─ Ingest multiple documents
├─ Batch process: for file in docs/*.md; do ingest...; done
└─ Chat against entire knowledge base

USE CASE 3: API Integration
├─ Query /query endpoint from your apps
├─ Embed inRAG context into your services
└─ Get answers with sources
